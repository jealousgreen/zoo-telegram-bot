# Telegram-бот викторина для зоопарка

Бот-викторина для Telegram-канала Московского зоопарка. Пользователь отвечает на 8 вопросов, получает тотемное животное, фото/ссылку на страницу животного, короткое описание и блок про программу опеки.

## Что реализовано

- `/start`, `/help`, `/privacy`.
- Дружелюбная викторина с inline-кнопками.
- Настраиваемые вопросы и веса в `src/bot/data/quiz.py`.
- Взвешенный алгоритм выбора животного.
- Финальный результат с описанием, топом совпадений, ссылкой на страницу животного и кнопками.
- Фото: бот сначала использует `photo_url`, если он задан у животного, иначе пытается взять изображение со страницы вида на `moscowzoo.ru/animals/kinds/...` через OpenGraph/Twitter/JSON-LD/`img`. Если сайт изменился или фото не найдено, бот отправит текстовый результат со ссылкой на страницу животного.
- Кнопки «Узнать про опеку», «Связаться с сотрудником», «Поделиться в Telegram/VK», «Попробовать ещё раз», «Оставить отзыв».
- Контактный механизм: если задан `ADMIN_CHAT_ID`, сотруднику приходит заявка с результатом викторины пользователя.
- Обратная связь: отзывы сохраняются в SQLite и пересылаются администратору.
- Минимальное хранение данных: Telegram ID, ответы, результат и отзыв.
- Логи в `logs/bot.log` и консоль.
- Тесты для алгоритма скоринга.

Большие панды не включены в вопросы и результаты.

## Быстрый запуск

```bash
git clone https://github.com/jealousgreen/zoo-telegram-bot.git
cd zoo_telegram_bot
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env .env
```

Заполните `.env`:

```env
BOT_TOKEN=ТОКЕН
BOT_USERNAME=username_бота
ADMIN_CHAT_ID=telegram_id_админа
```

Запуск:

```bash
PYTHONPATH=src python -m bot
```

После нажатия «Связаться с сотрудником» бот отправит администратору:

- имя пользователя;
- username, если есть;
- Telegram ID;
- итоговое животное викторины.


Структура ответа:

```python
AnswerOption("Текст ответа", {"animal_id": 3, "other_animal_id": 1})
```

Чем больше баллов получает животное, тем выше шанс стать итоговым результатом.

## Структура проекта

```text
zoo_telegram_bot/
├─ src/bot/
│  ├─ data/quiz.py              # животные, вопросы, веса ответов
│  ├─ handlers/common.py        # старт, справка, опека, privacy
│  ├─ handlers/quiz.py          # прохождение викторины и результат
│  ├─ handlers/contact_feedback.py
│  ├─ services/scoring.py       # алгоритм подсчёта
│  ├─ services/photo_provider.py# получение фото со страниц животных
│  ├─ services/storage.py       # SQLite
│  ├─ config.py                 # настройки
│  └─ main.py                   # точка входа
├─ tests/test_scoring.py
├─ requirements.txt
├─ .env
└─ README.md
```

## Масштабирование и безопасность

- Токен и служебные контакты не хранятся в коде.
- При росте нагрузки можно заменить `Storage` на PostgreSQL/Redis, не меняя логику хендлеров.
- Бот хранит минимум персональных данных.
- Ошибки логируются в файл и консоль.
- Фото не складываются в репозиторий: используются ссылки на официальные страницы видов или прямые URL, которые можно указать вручную.
